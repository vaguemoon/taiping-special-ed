# 朗讀趣 — 設計方案

## 核心定位

透過麥克風偵測學生朗讀課文，結合準確率與流暢度決定通過與否。
老師在後台指派段落與門檻，學生完成後系統自動評分並逐字標色。

---

## 決策總表

| 項目 | 決策 |
|------|------|
| 語音評估引擎 | Azure Speech Pronunciation Assessment（zh-TW） |
| 評分維度 | Accuracy + Fluency + CPM（字/分鐘）；Prosody 延後實作 |
| 比對邏輯 | 直接使用 Azure 原生 ErrorType，不加 Levenshtein Distance |
| API 金鑰安全 | Firebase Cloud Function 代理短效 token，金鑰不出現在前端 |
| 錄音模式 | 批次：錄完整段 → 送出 → 事後逐字上色（✅/❌） |
| 每次錄音上限 | 30 秒，超時自動截斷送出 |
| 月用量上限 | 全站 18,000 秒（5 小時），達限後停用評分功能；最高費用約 $18 USD/月 |
| 學生隱私 | 朗讀頁面顯示告知文字（聲音僅用於語音辨識，不儲存） |
| 段落資料 | 掛在現有 curriculum 課程下，新增 `passages` 欄位 |
| 學生選段 | 老師在班級設定中指派段落，學生只看到被指定的段落 |
| 重試次數 | 固定上限（預設 3 次），老師可調整；超過後段落鎖定 |
| 通過門檻 | 準確率 + 字速（CPM）雙門檻，老師分別設定 |

## Firestore 資料結構

```
curriculum/{ver}/lessons/{lessonId}
  ├── chars, words, questions    ← 現有欄位不動
  └── passages: [                ← 新增
        { id, title, text }
      ]

classes/{classId}
  └── readingAssignment: {       ← 新增
        lessonId, passageId,
        maxAttempts: 3,          ← 老師設定
        accuracyThreshold: 85,   ← 老師設定（%）
        cpmThreshold: 200,       ← 老師設定（字/分鐘）
        enabled: true
      }

students/{id}/progress/reading
  └── {
        {passageId}: {
          attempts: [{ date, accuracy, fluency, cpm,
                       wordErrors: { 字: count }, passed }],
          bestAccuracy, bestCpm, locked: bool
        },
        lastStudied
      }

system/usage
  └── { audioSeconds, monthKey: "2026-05", updatedAt }
```

## 檔案架構

```
apps/learn/lang/reading/
├── index.html
├── reading.css
└── js/
    ├── init.js        ← sessionStorage 登入 + 讀取班級指派段落
    ├── state.js       ← 進度讀寫、月用量 increment
    ├── nav.js         ← 頁面切換
    ├── passage.js     ← 段落文字顯示、事後逐字上色
    ├── recorder.js    ← MediaRecorder + 30 秒計時 + Azure token 取得
    ├── assessment.js  ← Azure 結果解析 → 字詞 ErrorType 映射
    └── achievement.js ← 成就計算（複用現有模式）

Firebase Cloud Functions（新增）
└── getAzureToken()
    ├── 檢查 system/usage，超過 18,000 秒回傳 403
    └── 換取 Azure 10 分鐘短效 token 回傳給前端
```

## 學生端流程

```
進入 App
  → 讀取班級指派的段落與剩餘次數
  → 顯示課文全文 + 錄音按鈕 + 隱私告知文字
  → 倒數 30 秒錄音（可提前停止）
  → 送出 → Cloud Function 取 Azure token → 送至 Azure 評分
  → 事後逐字上色（✅ 綠 / ❌ 紅）
  → 顯示準確率、CPM、是否通過雙門檻
  → 更新剩餘次數；達上限後段落鎖定
```

**UI 元件對應現有模式：**
- 指派欄位 → 沿用 `math-quiz.js` 的 numeric input + 下拉組合
- 成績列表 → 沿用 `quiz-sessions.js` 的 `<details>` 展開模式
- 用量進度條 → 沿用 `students.js` 的 progress bar CSS
- 統計數字 → 沿用 `admin.css` 的 `.stat-card` chips

### 課程資料頁籤（現有，新增子功能）

每課編輯介面下方新增「朗讀段落」子區塊：
- 可新增 / 刪除段落（每筆含標題 + 多行文字框）
- 儲存至 `curriculum/{ver}/lessons/{id}.passages[]`

## 備註

- **Prosody 評分**：Azure zh-TW 支援不穩定，列為未來功能
- **費用試算**：Azure Pronunciation Assessment 約 $1 USD / 1,000 秒，5 小時上限對應最高 $18 USD/月
- **實作時機**：預計 5 月 usage 充足後開始
