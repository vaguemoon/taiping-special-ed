# 乘法趣（multiply/）— 檔案功能說明

> 給未來 AI 閱讀的文件。說明每個 JS 檔的職責、依賴關係與重要設計決策。

---

## 目錄結構

```
multiply/
├── index.html          ← App 入口，含所有頁面 div 與 script 引用
├── multiply.css        ← App 專屬樣式（禁止寫死顏色，使用 CSS 變數）
├── 檔案功能說明.md      ← 本文件
└── js/
    ├── nav.js          ← 頁面切換、PAGE_STACK
    ├── state.js        ← 全域狀態、Firestore 進度存取
    ├── game.js         ← 練習模式 + 測驗模式完整邏輯
    ├── achievement.js  ← 成就系統
    └── init.js         ← 啟動、自動登入、Topbar 選單、設定頁
```

## js/nav.js

**職責：** 頁面切換與導覽堆疊

| 全域變數 | 說明 |
|---------|------|
| `PAGE_STACK` | 頁面歷史堆疊（陣列），`goBack()` 靠它決定上一頁 |
| `PAGE_CONFIG` | 各頁面的標題與是否顯示返回鍵 |
| `currentPage` | 目前所在頁面 ID |

| 函式 | 說明 |
|-----|------|
| `showPage(name, pushHistory)` | 切換頁面、更新 Topbar 標題與返回鍵、自動觸發成就渲染 |
| `goBack()` | 返回上一頁（pop PAGE_STACK） |
| `backToHub()` | 透過 postMessage 通知 Hub 返回 |

**注意：** `pushHistory=false` 用於「返回」動作，避免 PAGE_STACK 重複。

## js/state.js

**職責：** 所有共用全域變數、Firestore 進度讀寫

### 關鍵變數

| 變數 | 說明 |
|-----|------|
| `currentStudent` | `{ name, pin, id, nickname, avatar }` |
| `practiceType` | `'choice'` / `'fill'` / `'reverse'` |
| `practicePool` | 本次練習題目池 `[{a, b}]` |
| `practiceQ` | 當前練習題目 |
| `practiceSelectedA_temp / B_temp` | 設定頁暫存選擇（離開頁面後仍保留） |
| `examType` | `'fill'` / `'reverse'` |
| `examTimerSec` | 每題倒數秒數（5 / 8 / 10） |
| `examPool` | 本輪待答題目 |
| `examAllPairs` | 本次所有選定題目 |
| `examMastered` | 本次已答對的 pairKey 字串（`'3_7'`） |
| `examPendingWrong` | 本輪答錯，下輪繼續 |
| `examSelectedTables_temp` | 設定頁暫存選擇 |
| `fillInputStr` | 填空輸入暫存字串 |
| `masteredFill` | 已完成填空測驗的乘數（如 `['3','5']`） |
| `masteredReverse` | 已完成積的拆解測驗的乘數 |
| `totalCorrect / totalAttempts` | 累積答對/總答題數 |

### Firestore 路徑
- **進度**：`students/{id}/progress/multiply`
  - 欄位：`masteredFill`, `masteredReverse`, `totalCorrect`, `totalAttempts`, `lastStudied`

## js/game.js

**職責：** 所有遊戲邏輯（選項生成、練習流程、測驗流程、填空輸入）

### 選項生成
- `generateChoiceOptions(a, b)` → 4 個數字選項（1 正確 + 3 鄰近干擾）
- `generateReverseOptions(a, b)` → 4 個算式對選項（積各不相同）

### 練習模式
- `initPracticeSelect()` → 初始化練習設定頁 UI
- `startPractice()` → 驗證選擇後建題目池並進入練習頁
- `loadPracticeQuestion()` → 隨機從 practicePool 取一題
- `onPracticeResult(correct)` → 處理答題結果（更新連續答對、觸發成就）
- `onPracticeChoiceAnswer(val, correct, btn)` → 選擇題答案
- `onPracticeReverseAnswer(btn)` → 積的拆解答案
- `onPracticeFillSubmit()` → 填空題提交

### 測驗模式（多輪 + 倒數計時）
- `initExamSelect()` → 初始化測驗設定頁 UI
- `startExam()` → 建題目池並進入測驗頁（第 1 輪）
- `loadExamQuestion()` → 從 `examPool[0]` 取題，若空則 `showRoundSummary()`
- `startExamCountdown()` → 啟動倒數計時器（setInterval）
- `onExamTimeout()` → 時間到：標記錯誤、顯示正確答案、1.5 秒後下一題
- `onExamCorrect()` → 答對：加入 examMastered、更新進度、下一題
- `onExamWrong()` → 答錯：加入 examPendingWrong、1.5 秒後下一題
- `showRoundSummary()` → 顯示輪間結算面板（正確數、待複習數）
- `startNextRound()` → 開始下一輪（只含上輪答錯題目）
- `onExamComplete()` → 全部答對：更新 masteredFill/masteredReverse、觸發成就
- `showExamFinalResult()` → 跳到測驗結果頁

### 填空輸入
- `fillAppend(d)` / `fillBackspace()` → 操作 `fillInputStr`
- `updateFillDisplay()` → 更新 DOM 中 `#fill-display` 的顯示
- `handleFillKeydown(e)` → 鍵盤支援（數字鍵、Backspace、Enter）

## js/achievement.js

**職責：** 成就統計、解鎖判斷、頁面渲染

### Firestore 路徑
- **成就**：`students/{id}/stats/multiplyProfile`（與練字趣隔離）
  - 欄位：`stars`, `title`, `totalLoginDays`, `lastLoginDate`, `unlockedAchievements`

### 成就類別
| 類別 | 解鎖 key 格式 | 說明 |
|-----|-------------|------|
| 答對題數（10 等） | `correct_lv1` ~ `correct_lv10` | totalCorrect 達門檻 |
| 累積登入（10 等） | `login_lv1` ~ `login_lv10` | totalLoginDays 達門檻 |
| 精熟乘法填空 | `fill_table_0` ~ `fill_table_10` | masteredFill 含該乘數 |
| 精熟積的拆解 | `rev_table_0` ~ `rev_table_10` | masteredReverse 含該乘數 |
| 完全精熟 | `both_table_0` ~ `both_table_10` | 填空 + 拆解都精熟 |

### 重要函式
- `checkAchievements()` → 所有成就檢查 + recomputeStars + saveAchStats
- `checkTableAchievements()` → 乘法表三類成就解鎖邏輯
- `handleDailyLogin()` → 每日首次登入 +1 loginDay
- `loadAchStats(callback)` → 從 Firestore 載入，完成後 callback
- `renderAchievementPage()` → 渲染成就頁面（header、標準軌道、表格方塊）

## js/init.js

**職責：** 應用程式啟動、自動登入、Topbar 選單、設定頁邏輯

### 啟動流程（window.load）
1. `initFirebase()` → `applyTheme()` → `applySound()` → `initSoundWrapper()`
2. `showPage('home', false)` → `PAGE_STACK = ['home']`
3. 掛載鍵盤事件（`handleFillKeydown`）和點擊收折選單事件
4. 從 `sessionStorage('hub_student')` 自動登入 → 載入 progress/multiply
5. `loadAchStats()` → `handleDailyLogin()`

### Topbar 函式
- `toggleLogoutMenu()` → 顯示/隱藏下拉選單
- `sendLogout(evt)` → postMessage `multiply-logout` 給 Hub
- `goToSettings(evt)` → 渲染設定頁後跳轉
- `goToAchievement(evt)` → 跳轉成就頁（nav.js 會自動渲染）

### 設定頁
- `AVATARS` → 20 個表情符號頭像陣列
- `renderSettingsAvatarGrid()` → 渲染頭像選擇格
- `selectSettingsAvatar(av)` → 選頭像並寫入 Firestore

## 重要設計決策

1. **測驗計時**：採倒數計時（非正計時），時間到直接算錯。目的是訓練機械式反應，預設 8 秒。
2. **題型分離**：練習支援三種題型（選擇、填空、拆解），測驗只支援兩種（填空、拆解）。
3. **精熟定義**：需完成整個乘法表（11 題全對）才標記為精熟，與多輪設計配合。
4. **填空雙輸入**：同時支援畫面上的數字鍵盤和鍵盤輸入（`handleFillKeydown`）。
5. **成就隔離**：使用 `stats/multiplyProfile` 而非 `stats/profile`，避免與練字趣衝突。
6. **狀態持久**：`practiceSelectedA_temp`、`examSelectedTables_temp` 在切換頁面後仍保留，回到設定頁不需重選。
