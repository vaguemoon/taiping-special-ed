# chinese-quiz-app

> 國小國語科平板互動練習系統

---

## 專案主旨

本專案為針對國小國語課設計的數位練習工具，以**康軒版三年級與五年級**教材為初始內容，提供老師彈性出題、學生在平板上觸控作答的完整流程。

核心目標有三：

一是**降低出題門檻**，老師只需選擇年級與課次、輸入各題型題數，系統即自動從題庫隨機抽題產生試卷，無需手動排版。

二是**符合平板操作習慣**，所有互動以觸控為優先設計：填充題採拖曳詞卡方式作答，選擇題以點擊選項完成，避免學生需要打字輸入。

三是**可持續擴充的資料架構**，題庫儲存於 Firebase Firestore，老師可透過後台管理頁面隨時上傳新的 `.xlsx` 題庫，不需修改程式碼。

---

## 功能架構

### `quiz.html` — 學生作答頁

老師設定出題範圍（年級、課次、各題型題數）後產生試卷。試卷包含三種題型：

- **詞語解釋**：題目顯示詞語的解釋說明，學生將詞卡拖曳至對應空格
- **詞語填空**：句子中含有空格，學生將詞卡拖曳填入
- **選擇題**：點選 A / B / C 其中一個選項

作答完成後輸入姓名，點擊批改即顯示逐題對錯與總分，結果自動儲存至 Firebase。

### `admin.html` — 後台管理頁

提供兩項功能：Firebase 連線設定（一次性填入，儲存於瀏覽器本機），以及題庫 `.xlsx` 上傳介面。上傳時系統自動解析工作表內容並批次寫入 Firestore，支援預覽前十筆確認格式無誤後再送出。

### `records.html` — 成績紀錄頁

顯示所有作答紀錄，含作答總次數、不重複學生人數、平均分數、最高分等統計。支援依姓名搜尋、年級篩選、分數排序，每頁顯示二十筆並提供分頁導覽。

## 資料庫設計（Firebase Firestore）

### `questions` collection — 題庫

| 欄位 | 類型 | 說明 |
|---|---|---|
| `grade` | string | 年級，例：`康軒三上` |
| `lesson` | string | 課次，例：`七` |
| `lessonName` | string | 課名，例：`淡水小鎮` |
| `type` | string | 題型：`詞語解釋` / `詞語填空` / `選擇題` |
| `question` | string | 題幹文字 |
| `answer` | string | 正確答案 |
| `options` | array | 選擇題選項（非選擇題為空陣列） |
| `createdAt` | string | 上傳時間（ISO 8601） |

### `records` collection — 成績紀錄

| 欄位 | 類型 | 說明 |
|---|---|---|
| `name` | string | 學生姓名 |
| `score` | number | 得分（0–100） |
| `correct` | number | 答對題數 |
| `total` | number | 總題數 |
| `grade` | string | 作答年級 |
| `lesson` | string | 作答課次 |
| `timestamp` | string | 作答時間（ISO 8601） |

## 題庫 xlsx 格式規範

上傳的工作表須包含以下欄位名稱（第一列為標題列）：

| 欄位名稱 | 必填 | 說明 |
|---|---|---|
| `課次` | ✓ | 中文數字，例：七、八、十一 |
| `課名` | ✓ | 課文名稱 |
| `題型` | ✓ | `詞語解釋`、`詞語填空`、`選擇題` 三者之一 |
| `題幹` | ✓ | 題目文字；選擇題可用 `\|` 分隔選項 |
| `答案` | ✓ | 正確答案文字（選擇題填 A / B / C） |

選擇題題幹支援兩種格式：

```
問題文字|A. 選項甲|B. 選項乙|C. 選項丙
```

或直接內嵌於題幹：

```
底層主要放什麼？A. 沙子　B. 小石頭　C. 大竹子
```

## 子 App 設計準則（測驗類）

本專案屬於 QuEDU 平台的**測驗類子 App**，須遵循以下平台準則：

- **不提供成就系統**：測驗類 App 不設計勳章、等級或成就累積機制，聚焦於當次練習的即時批改與分數回饋。
- **視覺風格**：以「紙本練習卷數位化」為概念，使用米白底色（`#faf8f3`）搭配深墨色（`#1a1a2e`）模擬印刷質感，輔以細邊框與柔和陰影提供層次感，避免過度螢幕感的高彩配色。
- **字型**：選用 Google Fonts 的 **Noto Serif TC**（標題、題號）與 **Noto Sans TC**（內文、介面），確保中文字符完整顯示，在各尺寸螢幕上保持清晰。

## 設計方向

### 視覺風格

以「紙本練習卷數位化」為概念，使用米白底色（`#faf8f3`）搭配深墨色（`#1a1a2e`）模擬印刷質感，輔以細邊框與柔和陰影提供層次感，避免過度螢幕感的高彩配色。

字型選用 Google Fonts 的 **Noto Serif TC**（標題、題號）與 **Noto Sans TC**（內文、介面），確保中文字符完整顯示，在各尺寸螢幕上保持清晰。

### 互動設計

拖曳功能同時支援滑鼠（`mousedown / mousemove / mouseup`）與觸控（`touchstart / touchmove / touchend`），並以一個「浮動跟隨手指的詞卡幽靈元素」提供視覺回饋，讓學生清楚感知正在拖曳的詞語。

已填入空格的詞卡顯示 × 按鈕，允許清除後重新選擇，降低誤操作帶來的挫折感。

批改後逐題顯示綠色（正確）或紅色（錯誤）標示，並於錯誤題目揭示正確答案，強化即時學習回饋。

### 架構決策

三個頁面均為純 HTML 單檔，引用 CDN 資源（Firebase SDK、SheetJS），不需建置流程或本機伺服器，可直接以瀏覽器開啟或部署至任何靜態網頁托管服務（GitHub Pages、Firebase Hosting 等）。

`quiz.html` 內嵌初始題庫 JSON 作為離線備援，Firebase 連線成功時優先使用雲端資料；連線失敗時自動降級至本機資料，確保課堂中網路不穩時仍可正常使用。

## 開發計畫

### 整合目標

將 chinese-quiz 作為 QuEDU 平台的子 App，嵌入 Hub 主頁的「測驗區」，學生由 Hub 點入後以 iframe 開啟，作答完成後透過 `postMessage` 回到 Hub。

---

### 階段一：整合基礎架構

**目標**：讓現有的 `quiz.html`（學生作答頁）可在 Hub iframe 中正常啟動並回傳。

| 項目 | 說明 |
|---|---|
| Hub 傳遞學生資訊 | Hub 將 `hub_student`（JSON）透過 `sessionStorage` 傳入，quiz 頁面啟動時讀取姓名與學生 ID |
| 返回 Hub | 作答或批改完成後，呼叫 `window.parent.postMessage({ type: 'chinese-quiz-back-to-hub' }, '*')` |
| 成績寫入 Firestore | 批改後將 `{ app, score, correct, total, grade, lesson, timestamp }` 寫入 `students/{sid}/activities` |
| 入口頁 `index.html` | 新建一個輕量入口頁，讀取 sessionStorage 中的學生資訊後直接跳轉 `quiz.html`，並顯示返回按鈕 |

**交付物**：`chinese-quiz/index.html`（入口）、修改後的 `chinese-quiz/quiz.html`

---

### 階段二：出題設定介面

**目標**：讓學生在進入作答前，可以選擇年級與課次。

| 項目 | 說明 |
|---|---|
| 年級選擇 | 從 Firestore 的 `questions` 讀取所有不重複的 `grade` 值，以按鈕呈現 |
| 課次選擇 | 選定年級後，讀取對應 `lesson` 清單，以按鈕或下拉選單呈現 |
| 題數設定 | 各題型（詞語解釋、詞語填空、選擇題）分別設定抽題數，提供預設值 |
| 離線備援 | 若 Firestore 不可用，降級至內嵌的 `fallback-questions.js` |

**交付物**：`chinese-quiz/index.html` 加入設定 UI，或獨立為 `chinese-quiz/setup.html`

---

### 階段三：作答體驗優化（平板適配）

**目標**：強化拖曳填空與選擇題在平板上的操作流暢度。

| 項目 | 說明 |
|---|---|
| 拖曳同時支援觸控與滑鼠 | `touchstart/touchmove/touchend` + `mousedown/mousemove/mouseup` 雙軌實作 |
| 浮動幽靈詞卡 | 拖曳時顯示跟隨手指移動的半透明複製卡，增強視覺回饋 |
| 清除機制 | 已填入空格的詞卡顯示 × 按鈕，可清除重選 |
| 紙本視覺風格 | 套用米白底色、Noto Serif TC 題號、細邊框，符合測驗類準則 |
| 批改動畫 | 逐題顯示綠色（✓）/ 紅色（✗）標示，錯誤題揭示正確答案 |

**交付物**：更新後的 `chinese-quiz/quiz.html`、`chinese-quiz/chinese-quiz.css`

---

### 階段四：題庫管理（教師後台）

**目標**：教師可透過後台上傳 `.xlsx` 題庫，無需修改程式碼。

| 項目 | 說明 |
|---|---|
| 上傳 UI | 選擇 `.xlsx` 檔案後解析，顯示前十筆預覽，確認後批次寫入 Firestore |
| 格式驗證 | 檢查必要欄位（課次、課名、題型、題幹、答案）是否齊全，顯示錯誤提示 |
| 整合教師後台 | 考慮放入現有的 `teacher/` 後台，或維持獨立的 `chinese-quiz/admin.html` |

**交付物**：`chinese-quiz/admin.html`（或整合至教師後台）

---

### 階段五：成績紀錄與班級視圖

**目標**：教師可查看全班作答紀錄與統計。

| 項目 | 說明 |
|---|---|
| 紀錄頁 | 顯示所有紀錄，支援姓名搜尋、年級篩選、分數排序，每頁 20 筆 |
| 班級篩選 | 依 `classId` 篩選，只顯示本班學生 |
| 統計摘要 | 顯示作答次數、不重複人數、平均分、最高分 |

**交付物**：`chinese-quiz/records.html`

---

### 開發優先序

```
階段一（整合）→ 階段三（體驗）→ 階段二（出題設定）→ 階段四（題庫管理）→ 階段五（成績紀錄）
```

階段一是最小可用版本（MVP），可在不動現有功能架構的情況下讓 chinese-quiz 出現在 Hub 測驗區並可正常作答與回傳。
