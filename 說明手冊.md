## 一、整體架構概覽

```

├── index.html          ← 入口頁（登入 + 科目選單）

    └── js/

        ├── state.js        ← 全域資料 + Firestore 讀寫
        ├── voice.js        ← 文字轉語音
        ├── curriculum.js   ← 課程選擇
        ├── menu.js         ← 字卡主畫面

        └── init.js         ← 啟動初始化
```

**頁面關係圖：**
```

admin.html   (獨立，不使用 iframe)
```

### `index.html` — 入口頁 / 學生 Hub

**職責：** 整個系統的起點。處理學生登入、顯示科目選單、載入各科目 app。

**包含四個畫面（panel）：**

| 畫面 id | 功能 |
|---|---|
| `#panel-login` | 學生登入 / 註冊（姓名 + 4 位 PIN） |
| `#panel-hub` | 科目選單（目前只有「國字練習」） |
| `#panel-subject` | `<iframe>` 容器，用來載入各科目 |
| `#panel-profile` | 個人設定（暱稱、頭像、主題色、音效） |

**主要功能：**
- **登入邏輯**：學生輸入姓名 + PIN，查詢 Firestore `students/{name}_{pin}`。第一次使用會自動建立帳號。
- **教師登入**：畫面底部隱藏入口（🔑 教師登入），密碼 `381418`，驗證後導到 `admin.html`。
- **科目切換**：點擊科目卡片，將該科目的 HTML 載入 `<iframe>`，並透過 `sessionStorage` 傳遞學生資料。
- **自動登入**：若 `sessionStorage` 已有學生資料，直接跳到 Hub，不需重新登入。
- **最近活動**：Hub 畫面顯示最近練習過的字（已精熟/練習中）。

**修改此檔案時機：**
- 新增/修改科目卡片
- 修改登入畫面外觀
- 修改 Hub 主頁版面
- 修改個人設定項目

### `shared.js` — 共用 JavaScript 函式庫

**職責：** 所有頁面的基礎層，提供跨頁共用功能。

**提供的功能模組：**

#### 1. Firebase 初始化
```
initFirebase()
```
初始化 Firebase app（專案：`tainping-hanzi-app`），建立全域 `db` 物件供其他 JS 使用。

#### 2. 主題系統
```
THEMES[]  /  applyTheme(themeId)  /  renderThemeGrid()
```
六個主題：藍天、森林、薰衣草、夕陽、青空、粉紅。  
覆蓋 CSS 變數 `--blue`、`--bg` 等。選擇存入 `localStorage`。

#### 3. 音效開關
```
soundEnabled  /  applySound()  /  toggleSound()
```
存入 `localStorage`，控制全站音效播放。

#### 4. Toast 通知
```
showToast(msg)
```
在畫面底部顯示短暫提示訊息（需頁面有 `<div id="toast">`）。

#### 5. 頂部導覽列
```
renderTopbar(target, opts)
```
動態注入標準化頂部導覽列，支援返回按鈕、標題、設定齒輪按鈕。

#### 6. Web Audio 音效引擎
```
sfxTap()  /  sfxSwipe()  /  sfxCorrect()  /  sfxWrong()
sfxPass()  /  sfxCelebrate()  /  sfxGrandCelebrate()
```
用 `AudioContext` 合成音效（無外部音訊檔案）。支援 iOS AudioContext 解鎖。

#### 7. 頁面切換
```
goTo(toId)
```
切換 `.screen` 元素並播放左右滑入動畫。

#### 8. 子頁面初始化
```
initSubPage(callback)
```
科目模組的標準啟動流程：Firebase 初始化 → 套用主題 → 確認 `hub_student` 存在 → 呼叫 callback。

**修改此檔案時機：**
- 新增全站共用功能
- 修改主題顏色
- 修改音效
- 修改 Firebase 設定

### `admin.html` — 教師後台

**職責：** 教師管理介面，完全獨立的單一檔案。

**存取控制：** 進入時檢查 `sessionStorage.adminAuth === '381418'`，否則導回 `index.html`。

**兩個主要分頁：**

#### 分頁 1：班級總覽
- 統計列：學生總數、今日活躍人數、平均精熟字數
- 學生名單表格（依最後登入排序）
- 點擊學生：顯示該生詳細課程進度（按課本/課次分組）
- **學生詳細頁**：可查看 PIN、手動修改每個字的狀態（精熟/練習中/未學）、刪除學生

#### 分頁 2：課程管理
- 版本列表：列出所有教材版本，可展開查看課次
- **新增版本**：建立新的課程版本
- **CSV 批次匯入**：上傳 CSV（欄位：`版本,年級,課,課名,生字`），一次匯入大量課程資料
- **手動新增課次**：表單填寫版本名、年級、課次編號、課名、生字
- **刪除功能**：刪除版本或個別課次

**Firestore 讀寫範圍：**
```
students/{id}               讀取 / 修改狀態 / 刪除
students/{id}/progress/hanzi   讀取 / 修改 charStatus
curriculum/{verId}/lessons/    讀取 / 新增 / 刪除
settings/lesson             寫入（推送指定課文給所有學生）
```

**修改此檔案時機：**
- 修改管理後台版面
- 新增教師功能
- 修改學生資料管理邏輯

### `chinese/js/state.js` — 全域資料與 Firestore 讀寫

**職責：** 整個國字 app 的資料中心。

**儲存的資料：**

| 變數 | 說明 |

**Firestore 寫入：**
- `saveProgress()`：寫入 `students/{id}/progress/hanzi`（`charStatus` + 時間戳）
- `saveStroke(char, strokes)`：寫入 `students/{id}/strokes/{char}`（正規化筆畫座標）

### `chinese/js/menu.js` — 字卡主畫面

**職責：** 渲染字卡格、進度條、考試模式控制。

**主要功能：**
- `renderMenu()`：重新渲染所有字卡（顯示字、狀態徽章、精熟打勾），並自動呼叫 `saveProgress()`
- `updateProgressBar()`：計算三種狀態的字數比例，更新進度條寬度；全部完成時啟動考試按鈕閃爍

- **考試選字模式**：`enterExamSelectMode()` 讓學生點選要考的字；`startSelectedExam()` 只考選到的字
- `nextChar()`：切換到下一個字；全部完成時回到主畫面並播放慶祝音效

**修改此檔案時機：**

### `chinese/js/practice.js` — 筆順練習 + 單字考試

**職責：** HanziWriter 驅動的筆順練習和單字考試模式。

**兩大功能區塊：**

#### 練習模式（`switchToPractice()`）
- `panel-practice` 中顯示兩個 HanziWriter 畫布：
  - `#ref-target`（`refWriter`）：自動播放筆順動畫（參考用）
  - `#quiz-target`（`quizWriter`）：讓學生依筆順互動寫字
- `startQuiz()` 回呼：
  - 寫錯：紅色閃爍 + 錯誤音效 + 提示文字
  - 單筆正確：綠色閃爍 + 正確音效
  - 完成：狀態升為 `'practiced'`，顯示「開始默寫測驗」按鈕

#### 單字考試模式（`switchToSingleExam()`）
- 不顯示字形輪廓（`showOutline: false`），從記憶中寫
- 完成後判斷：
  - 0 錯誤 → `'mastered'` + 大慶祝音效
  - 1–3 錯誤 → `'practiced'`
  - 4+ 錯誤 → 狀態不改變

**修改此檔案時機：**
- 修改錯誤/正確的回饋方式
- 修改精熟條件（目前是 0 錯誤）
- 修改 HanziWriter 外觀設定（顏色、大小）

### `chinese/js/dictation.js` — 手寫默寫

**職責：** 自由手寫的默寫模式（不使用 HanziWriter，純 canvas 手繪）。

**運作流程：**
1. `switchToDict()`：顯示 `panel-dict`，初始化畫布
2. `initDictCanvas()`：設定畫布大小，綁定滑鼠和觸控事件
3. 學生自由書寫（記錄筆畫座標）
4. `submitDictation()`：截圖比對 — 並排顯示「你寫的」vs「正確字形」
5. 學生自評：
   - 「⭕ 像！」→ 狀態升為 `'practiced'`，儲存筆畫資料
   - 「✖ 不像」→ 顯示失敗回饋，可重試

**技術細節：**
- 筆畫座標正規化到 [0,1] 範圍後存入 Firestore
- 使用 `getImageData` 檢查是否有墨水（避免空白送出）

**修改此檔案時機：**
- 修改畫布筆觸樣式（顏色、粗細）
- 修改自評介面
- 修改儲存筆畫的邏輯

### `chinese/js/exam.js` — 多輪正式考試引擎

**職責：** 最複雜的模組。執行完整考試，自動處理多輪重考機制。

**考試流程：**

```
startFullExam()
    ↓ 打亂字序
showExamQuestion()   ← 顯示一題（HanziWriter，不顯示輪廓）
    ↓ 播放語音
startExamQQuiz()     ← 附加回呼
    │ 連續錯 3 次 → 顯示輪廓提示
    │ 總錯 6 次   → 改為描字模式（記錄為失敗）
    ↓
recordAndNext()      ← 記錄結果、更新狀態
    ↓ 還有未過的字？
showRoundSummary()   ← 顯示本輪通過/未通過
    ↓ 按「開始第 N 輪」
doStartNextRound()   ← 只重考未通過的字
    ↓ 全部通過
showExamFinalResult() ← 顯示最終成績（🏆/🌟/😊/💪）
```

**判斷標準：**
- 0 錯誤 → `'mastered'` + 大慶祝
- 1–3 錯誤 → `'practiced'` + 慶祝
- 4+ 錯誤 → 進入下一輪重考

**修改此檔案時機：**
- 修改考試判斷標準（幾錯算過/不過）
- 修改輪數機制
- 修改考試結果頁版面
- 修改提示出現的時機

---

### `chinese/js/init.js` — 應用程式啟動

**職責：** 整個國字 app 的啟動入口，協調所有模組初始化。

**啟動流程（`window.onload`）：**
1. `initFirebase()`、`applyTheme()`、`applySound()`
2. 等待 HanziWriter CDN 載入（`waitHW()` 輪詢）
3. 顯示 `page-mode-select`、重置歷史堆疊
4. 從 `sessionStorage` 讀取 `hub_student`
5. 從 Firestore 載入：學生資料、學習進度、教師推送課文
6. 若教師有指定課文（`settings/lesson`），自動帶入生字並開始
7. 顯示歡迎 Toast（「歡迎回來，{暱稱}！」）
8. 載入課程版本（`loadCurriculumVersions()`）

**修改此檔案時機：**
- 修改啟動順序
- 修改教師推送課文的邏輯
- 新增啟動時要讀取的 Firestore 資料

## 四、資料庫結構（Firestore）

```
students/{name_PIN}
    .name         學生姓名
    .pin          4 位 PIN 碼
    .nickname     暱稱
    .avatar       頭像 emoji
    .createdAt    建立時間
    .lastSeen     最後登入時間

    progress/hanzi
        .charStatus   { '字': 'new' | 'practiced' | 'mastered' }
        .lastStudied  最後練習時間

    strokes/{char}
        .strokes      正規化筆畫座標 [[x,y], ...]
        .updatedAt    更新時間

curriculum/{versionId}
    .name         版本名稱（如「翰林版」）

    lessons/{lessonId}
        .grade      年級（如 1）
        .lessonNum  課次（如 3）
        .name       課名（如「大家來上學」）
        .chars[]    生字陣列

settings/lesson
    .chars[]      教師目前指定的生字（推送給所有學生）
```
